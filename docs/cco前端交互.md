# CCO 前端交互设计

基于 `docs/claude code 场景分析.md` 中的数据模型，设计前端需要展示的交互形式。

---

## 页面结构

```
侧边栏:
  ├─ 概览 (Overview)         ← 已有，保留
  ├─ 会话列表 (Sessions)     ← 重新设计
  └─ 会话详情 (Session Detail) ← 新增核心页面
```

---

## 一、会话列表页 `/sessions`

### 展示内容

每个会话卡片：

```
┌──────────────────────────────────────────────────┐
│ 修复登录页面的样式问题                 2026-03-26 │  ← session_name + 日期
│                                                  │
│ 对话轮次: 8    子智能体: 3    压缩次数: 1        │  ← 统计概要
│ Token: 125.3K   费用: $0.42   耗时: 5m 30s       │
│                                                  │
│ 模型: opus-4-6                                   │
└──────────────────────────────────────────────────┘
```

### 交互
- 点击卡片 → 进入会话详情页
- 支持按日期筛选、按费用排序

## 二、会话详情页 `/sessions/:id` （核心页面）

展示**完整对话流程**：主智能体对话、工具调用、子智能体交互全部在一个时间线中。

### 2.1 页面布局

```
┌─────────────────────────────────────────────────────────────┐
│ ← 返回    修复登录页面的样式问题                             │
│           Session: a670867f    费用: $0.42    Tokens: 125.3K │
├──────────────────────────────────────────┬──────────────────┤
│                                          │                  │
│  对话流（主区域）                         │  侧边信息面板     │
│                                          │                  │
│  [User] 帮我修复登录页面的样式问题        │  会话信息         │
│                                          │  Token 统计       │
│  [Assistant] 让我先查看相关文件。         │  费用统计         │
│                                          │  智能体树         │
│  ┌ Glob { "src/**/*login*" } ┐           │                  │
│  │ → src/pages/Login.tsx     │           │                  │
│  │ → src/styles/login.css    │           │                  │
│  └───────────────────────────┘           │                  │
│                                          │                  │
│  ┌ Read { "src/pages/Login.tsx" } ┐      │                  │
│  │ → (文件内容，可折叠)           │      │                  │
│  └────────────────────────────────┘      │                  │
│                                          │                  │
│  [Assistant] 让我搜索相关组件。           │                  │
│                                          │                  │
│  ┌─ 🤖 Agent [Explore] ─────────┐       │                  │

│  │ (可展开的子Agent对话)         │       │                  │

│  │ Tokens: 3.2K  $0.005         │       │                  │
│  └───────────────────────────────┘       │                  │
│                                          │                  │
│  [Assistant] 已修复，修改了以下文件...     │                  │

│  [User] 再帮我看看注册页面               │                  │
│  ...                                     │                  │
│                                          │                  │
└──────────────────────────────────────────┴──────────────────┘
```

### 2.2 对话流中的元素类型

| 元素 | 数据来源 | 展示方式 |
|------|---------|---------|
| **用户消息** | messages 中 role=user 的文本内容 | 气泡，显示文本 |
| **助手消息** | messages 中 role=assistant 的 text blocks | 气泡，Markdown 渲染 |
| **思考过程** | response content 中 type=thinking | 可折叠区域，灰色斜体 |
| **普通工具调用** | tool_use + tool_result（通过 id 匹配） | 内联卡片：工具名 + 参数 + 结果（可折叠） |
| **并行工具调用** | 同一 response 中多个 tool_use | 并排排列的工具卡片 |
| **子智能体调用** | tool_name=Agent 的 tool_use | **可展开的嵌套对话流** |
| **上下文压缩** | session_history_list 的分界点 | 时间线分隔条 |

### 2.3 子智能体展示（核心交互）

子智能体和普通工具的关键区别：普通工具只有入参+出参，子智能体有**完整的对话过程**。

**折叠状态（默认）：**
```
┌─ 🤖 Agent [Explore]  "搜索所有引用了 LoginForm 的文件" ──┐
│ 状态: ✅ 已完成    Tokens: 3.2K    费用: $0.005           │
│ 工具调用: Grep × 2, Read × 1                             │
│                                            [展开详情 ▼]   │
└──────────────────────────────────────────────────────────┘
```

**展开状态：**
```
┌─ 🤖 Agent [Explore]  "搜索所有引用了 LoginForm 的文件" ──┐
│ 状态: ✅ 已完成    Tokens: 3.2K    费用: $0.005           │
│                                            [收起详情 ▲]   │
│                                                          │
│  ┌─ 子Agent对话流 ────────────────────────────────────┐  │
│  │                                                    │  │
│  │  [Explore] 让我搜索引用了 LoginForm 的文件          │  │
│  │                                                    │  │
│  │  ┌ Grep { pattern: "LoginForm" } ┐                 │  │
│  │  │ → src/App.tsx:15              │                 │  │
│  │  │ → src/routes.tsx:8            │                 │  │
│  │  └───────────────────────────────┘                 │  │
│  │                                                    │  │
│  │  [Explore] 找到 2 个文件，让我查看内容              │  │
│  │                                                    │  │
│  │  ┌ Read { file: "src/App.tsx" }  ┐                 │  │
│  │  │ → (文件内容，折叠)            │                 │  │
│  │  └───────────────────────────────┘                 │  │
│  │                                                    │  │
│  │  [Explore] 总结: 找到 3 个文件引用了...             │  │
│  │                                                    │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│ → 返回给主Agent的结果: "找到以下文件引用了 LoginForm..."  │
└──────────────────────────────────────────────────────────┘
```

### 2.4 并行工具调用展示

同一 response 中多个 tool_use → 水平并排：

```
[Assistant] 让我同时查看这两个文件。

┌─ Read ─────────────────┐  ┌─ Read ─────────────────┐
│ src/pages/Login.tsx     │  │ src/styles/login.css   │
│ → (内容，折叠)          │  │ → (内容，折叠)          │
└─────────────────────────┘  └─────────────────────────┘
```

### 2.5 上下文压缩展示

压缩不是对话消息，而是时间线上的分隔事件：

```
  [User] 最后一个问题...
  [Assistant] 回答...

  ═══════════════════════════════════════════════
   🗜️ 上下文压缩  |  归档至历史 #1  |  [查看归档]
  ═══════════════════════════════════════════════

  [User] (压缩后的新对话开始)
  ...
```

点击"查看归档" → 弹窗/抽屉展示该段被压缩前的完整对话（来自 session_history_list）。

## 五、数据流

### 需要的 API 接口

```
GET /api/sessions
  → [{ session_id, session_name, stats, created_at }]

GET /api/sessions/:id
  → {
      session_id, session_name,
      main_agent: {
        current_messages,
        tools_call_result_map,
        session_history_list,
        sub_agents: [
          {
            agent_id,              // = tool_call_id，天然关联父 agent 的 tool_use
            prompt,                // Agent 工具调用的 prompt 参数
            is_finished,           // 子智能体是否已完成任务
            current_messages,
            tools_call_result_map,
            sub_agents: [...]   ← 递归嵌套
          }
        ]
      }
    }

GET /api/stats
  → 概览统计（扩展子智能体和工具使用统计）
```

### 前端渲染逻辑

```
renderConversation(agent):
  遍历 agent.current_messages:
    对每条 message:
      role=user:
        如果包含 tool_result → 查找对应 tool_call，回填结果
        否则 → 渲染用户消息气泡

      role=assistant:
        遍历 content blocks:
          type=text → 渲染 Markdown 文本
          type=thinking → 渲染折叠的思考过程
          type=tool_use:
            查找 tools_call_result_map[id]:
              tool_name != "Agent":
                → 渲染普通工具卡片（入参 + 结果）
              tool_name == "Agent":
                → 找到 sub_agents 中 agent_id 匹配的子智能体
                → 渲染子智能体卡片（折叠/展开）
                → 展开时: 递归调用 renderConversation(sub_agent)

  在 session_history_list 分界点插入压缩分隔条
```
